# ADR 0003：简体优先与底本注取消

- 状态：已采纳（Accepted）
- 日期：2026-07-09
- 关联 issue：#28、#31
- 取代：ADR 0001 决策第 4 条（繁体权威、构建期 t2s 派生简体）及第 5 条（底本不在诗页逐首标注）中的 per-poem 底本注条款

## 背景

站点第一版以繁体录入正文，并计划构建期 `t2s` 派生简体（ADR 0001、#22）。上线后核对发现：现有正文并非严格出自可信底本，与「以中华书局点校本为准」的立意相悖。

改以本地中华书局《古诗源》epub（中华国学文库版，**简体**）为正文权威来源（见 #28）。字形决策需同步翻转：站点转为**简体优先**；界面元数据（卷名、站点描述等）与正文一致，全站不再出现繁体界面文字。

ADR 0001 第 5 条已规定底本在站点层面统一标注、诗页不逐首标注；随着 per-poem frontmatter `base` 字段在内容重导入中失去维护价值，正式取消该字段。

## 决策

1. **正文权威来源改为中华书局《古诗源》epub（中华国学文库版，简体）。** 卷目、选目、异文校勘仍以中华书局点校本为准；录入字形与 epub 一致，站点**简体优先**。

2. **推翻 ADR 0001 第 4 条（繁体权威、t2s 派生）。** 不再以繁体为权威底本字形，不再计划构建期 `t2s` 从繁体派生简体。简繁切换（LanguageToggle 占位）保持不实现；未来若做，方向反转为「简体权威、派生繁体」。

3. **取消 per-poem 底本注（`base` frontmatter）。** 底本唯一、在站点层面（README / 关于页 / 页脚）说明即可；诗页仅在确有异文时展示「异文/校记」。诗歌加载模块不再解析或暴露 `base` 字段。

4. **界面元数据简体化。** `volumes.json` 卷名、站点描述等 UI 可见文案一律简体（如 汉 / 魏 / 晋 / 宋 / 齐 / 梁 / 陈 / 北朝 / 隋）。

5. **字体子集管线不变。** 仍用霞鹜文楷（LXGW WenKai）；`collectSiteFontGlyphs` 自动从内容收集码点，字形集合随内容自动变化，不为此决策改动子集脚本。

## 后果

- 正面：站点定位、界面与正文字形一致；底本说明收敛到站点级，减少 frontmatter 冗余；agent 与维护者不再按「繁体权威」旧决策工作。
- 负面：正文重导入（#30）完成前，存量繁体正文与新版决策并存，需后续 ticket 消化；简繁切换若重启须重新设计派生方向。
- ADR 0001 其余条款（权威底本、卷目结构、不收诗经楚辞、站点级底本标注原则）仍然有效。
