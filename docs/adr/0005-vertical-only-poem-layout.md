# ADR 0005：取消横排与两套竖排正文

- 状态：已采纳（Accepted）
- 日期：2026-07-09
- 关联 issue：#35

## 背景

《古诗源》收录五言、七言齐言诗，也有四言、三言、乐府杂言、骚体和长篇歌行。旧诗页同时维护横排与竖排，竖排又统一按「奇数句三句一列、偶数句两句一列」分组，导致杂言长句容易撑出失衡长列，且正文句号、顿号原样出现在竖排中，不合古籍阅读体例。

横排/竖排双模式也增加了交互、文案、localStorage 偏好和注解浮层方向的维护成本。

## 决策

1. **诗页正文只保留竖排。** 删除横排渲染分支、阅读方向切换按钮和 `gushiyuan-reading-direction` 偏好。页面 chrome（面包屑、站点顶栏、上一首/下一首）仍保持横排。

2. **体例分类自动推导。** 全诗所有章、所有句去掉「。」「、」后字数相同，且字数为三至八字时，判为齐言；其中 **≤60 句的短篇齐言**进入「一句一列」。超过 60 句的长篇，即使仍是五言、七言齐言，也进入「栏内回行」。frontmatter 可选 `verticalLayout` 覆盖，取值仅为「一句一列」或「栏内回行」。

3. **短篇齐言使用「一句一列」。** 视觉顺序从右往左为标题、题署、正文；每句独立一列，章与章之间加大列距。小诗看格律，列断即句断。

4. **杂言与长篇使用「栏内回行」。** 每章一个固定栏高的 `vertical-rl` 容器，章内句子连续渲染为 inline 节点，由浏览器在容器装满后自动向左回行。诗句可以跨列，源流注解仍挂在原句节点上；章与章之间留出隔列空白，保留情节分段。

5. **竖排显示层去标点。** 正文 Markdown 不改，仍以句号作为内容规范和句子切分依据；渲染竖排时移除「。」「、」。简体和繁体走同一条去标点路径。

6. **源流浮层固定向左。** 注解浮层不再随阅读方向切换，统一从左侧弹出，避免遮挡右侧尚未读到的内容。

## 后果

- 正面：诗页交互更简单；短篇齐言节奏更清楚；杂言、长篇不再由单句长度或句数撑破版面；竖排体例统一无标点。
- 负面：横排阅读入口完全移除；超过 60 句的齐言长诗不再逐句展示格律，而是优先保证古籍版面连续阅读。
- 维护：分类、覆盖解析、去标点和滚动定位集中在 `src/lib/vertical-layout.ts`，由纯函数单测和内容校验覆盖；逐卷审校只需为误判诗补 `verticalLayout`。
