# ADR 0001：权威底本与选本体例

- 状态：已采纳（Accepted）；决策第 4、5 条字形/per-poem 底本注条款已由 ADR 0003 取代
- 日期：2026-07-08
- 关联 issue：#26、#22

## 背景

站点「古诗源」此前把书名当作「唐前古诗」的泛称，按朝代粗分卷目（先秦 / 汉 / 魏 / 晋 / 宋齐梁陈 / 隋），并在「先秦」卷下混入《诗经》篇目（关雎、蒹葭、桃夭）。这与沈德潜《古诗源》原书的编选体例直接冲突——原书是一部**具体的书**（清·沈德潜选评，十四卷，录诗七百余首），目录第一条为「古逸」，且明确不收《诗经》《楚辞》（以《诗经》为经、自成一系；《楚辞》另立）。

「古诗源」是什么，决定了目录结构、选目边界、底本校勘与简繁呈现，因此需要先确立权威底本与选本体例。

## 决策

1. **权威底本：中华书局点校本《古诗源》。** 卷目、选目、异文校勘一律以它为准。（曾考虑康熙竹啸轩初刻本；改用点校本，因其卷目清晰、校勘规范、易获取。）

2. **卷目以原书朝代段为纲，首条为「古逸」。** `volumes.json` 重构为 10 段：古逸 / 汉 / 魏 / 晋 / 宋 / 齐 / 梁 / 陈 / 北朝 / 隋。原书十四卷按朝代归并，`nan-bei`（宋齐梁陈）拆开，`xian-qin`（先秦）由「古逸」取代。

3. **不收《诗经》《楚辞》。** 关雎、蒹葭、桃夭等从正目剔除；其连带源流线索一并删除（源流讲「《古诗源》选诗如何被后世化用」，诗经不在选目内）。

4. **内容以繁体录入。** 繁体为权威底本字形。简体由构建期 `t2s` 从繁体派生（见 #22），运行期零转换；这从根上规避简→繁一对多翻车（云/雲、发/發/髮、里/裡/裏 等）。

5. **底本不在诗页逐首标注。** 底本在站点层面（README / 关于页 / 页脚）统一标注一次；诗页仅在确有异文时展示「异文/校记」。

## 后果

- 正面：站点定位明确指向沈德潜《古诗源》这本书；目录与选目与原书对齐；繁体为底本使简繁切换可安全派生。
- 负面：晋 / 宋 / 齐 / 梁 / 陈 / 北朝 / 隋 七卷暂为空，首页显示「整理中」，需后续补内容；古逸卷先补入 4 首种子（击壤歌、越人歌、易水歌、卿云歌）。
- 字体子集需覆盖繁体码点（由 `collectSiteFontGlyphs` 自动从内容收集）。

## 简繁切换（暂缓）

**决策（2026-07-08）：简繁切换功能暂不实施，#22 延期。** 站点定位与选目尚未稳定（七卷仍空、内容仅 15 首），简体无用户需求证据，且「只有繁体」当前是逼紧繁体录入质量的正面约束。

接口保留：繁体为权威底本的决策不变，将来若重启简繁，按「繁体零转换、简体由构建期 `t2s` 派生」落地即可，底本无需改动。待内容成熟（各卷补齐、有简体需求证据）时重启 #22。
