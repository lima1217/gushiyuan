# 古诗源 · 领域上下文

本文件是站点的单一领域上下文（single context），记录关键领域词汇与既定决策。决策细节见 `docs/adr/`。

## 底本（base edition）

**权威底本：中华书局点校本《古诗源》（中华国学文库版 epub，简体）。**

- 「古诗源」指清·沈德潜选评的具体选本（十四卷，录诗七百余首），不是「唐前古诗」的泛称。
- 卷目、选目、异文校勘一律以中华书局点校本为准（见 ADR 0001）；正文字形与 epub 一致，站点**简体优先**（见 ADR 0003）。
- 底本在站点层面（README / 关于页 / 页脚）统一标注一次；诗页不逐首标注底本，仅在确有异文时展示「异文/校记」。

## 标点体例（punctuation）

正文分句一律用句号「。」；逗号、冒号、引号、叹号、问号不出现在正文中。实体书原有的句内顿号「、」保留（已知仅《杖铭》《越人歌》两处）。见 #28。

## 排版（layout）

- 站点正文只保留**竖排**；页面 chrome（面包屑、上一首/下一首、站点顶栏）保持横排。
- **短篇齐言（≤60 句）**：全诗去掉「。」「、」后每句字数相同，且为三言至八言，采用「一句一列」。小诗看格律，列断即句断。
- **长篇叙事诗（＞60 句）与杂言、长句、骚体、长篇歌行**：采用「栏内回行」，每章进入固定栏高的竖排容器，文字装满一列后自动向左接续；即使五言、七言齐言，只要篇幅巨大也不一句一列。
- 两套正文竖排版式都不显示「。」「、」；多章诗每章重新起列，回行版式用章间留白保留分段。体例分类自动推导，可用 frontmatter `verticalLayout` 覆盖为「一句一列」或「栏内回行」。

## 古逸（guyi）

《古诗源》卷一，收上古至先秦的逸诗、歌谣、谚语（击壤歌、越人歌、易水歌、卿云歌等）。是原书目录的**第一条**，体现沈德潜「《诗经》《楚辞》之外、上探唐诗之源」的编选立意。站点 `volumes.json` 首卷 `slug: gu-yi`。

## 卷（volume）

站点按原书朝代段归并分卷，共 10 段：古逸 / 汉 / 魏 / 晋 / 宋 / 齐 / 梁 / 陈 / 北朝 / 隋（对应原书十四卷）。路由 `/v/[volumeSlug]`。卷下按诗人 → 诗 → 阅读页。卷名等界面元数据为简体。

## 源流（lineage）

源流讲《古诗源》**选诗**如何被后世化用。每条线索（clue）指向一首选诗的某句，并列出后世承袭的「流」（streams）。三种关系：`化用` / `脱胎` / `意象承接`。诗经楚辞不在选目内，故不作为源流源头。

源流是站点**唯一的行内注解形态**：带源流线索的诗句渲染为可点击/可聚焦的按钮（标记「源」，约 0.85em），点击打开 Popover 浮层（点击外部或 Esc 关闭），键盘与触屏均可用。浮层模式由通用组件 `AnnotationPopover` 提供（见 ADR 0002）。

## 阅读方向（reading direction）

阅读方向已固定为竖排，列自右向左；不再提供横排/竖排切换，也不再持久化 `gushiyuan-reading-direction`。注解浮层固定从左侧弹出。

## 简繁切换（script variant）

简体 Markdown 是唯一权威内容源；繁体由构建期 OpenCC s2t 派生，并叠加 `content/script-conversion-overrides.json` 的古文语境覆盖表。客户端只在已内嵌的简体/繁体文本之间切换，偏好存 localStorage，key `gushiyuan-script-variant`；切换时 `<html lang>` 同步为 `zh-CN` / `zh-Hant`。见 ADR 0004。

## 字形演变（已移除）

字形演变功能（`/c/[char]`、`content/characters/*`、`public/characters/*`）已整体移除（见 #18、ADR 0002）：字形数据不可靠且非核心价值。其点击 Popover 交互模式抽为通用组件 `AnnotationPopover`，迁用于「源」注解。移除后诗句按纯文本渲染，不再有关键字高亮与字形浮层。
